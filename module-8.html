Module 08

Technology & systems

By the end of this module you'll be able to
  • Map the systems a distributor runs — ERP, dispatch, telematics, tank monitoring, CRM, accounting — and what each owns.
  • Identify where the data lives, where it's siloed, and where the handoffs break.
  • Spot the integration points where automation creates the most leverage on the four controllable levers.

Recall the four controllable levers from Module 1: which rack you buy from, how you route trucks, how you price bids, and how fast you collect cash. Technology is how a distributor actually pulls those levers at scale. The problem in most distributors isn't a lack of systems — it's that the systems don't talk to each other, so data is re-keyed, delayed, or lost between them.

The typical stack, layer by layer

  • ERP (the system of record) — accounts, pricing, contracts, order history, inventory, billing. Modern fuel ERPs (e.g. iRely i21) aim to unify this. The ERP is the spine; if it's incomplete or being migrated to, everything downstream suffers.
  • Dispatch / routing — turns orders into truck routes and schedules (e.g. a "digital dispatcher"). This is where route density (Module 3) is won or lost operationally.
  • Tank monitoring / telematics — sensors on customer tanks reporting fuel levels, and GPS/telematics on trucks reporting location and delivery data. The source of real-time demand and logistics signal.
  • CRM / sales — pipeline, accounts, and bids (e.g. SPOTIO). Where new volume is tracked before it becomes an order.
  • Accounting — the books, A/R, A/P (e.g. QuickBooks). Where cash collection (the fourth lever) actually happens.
  • Legacy / interim tools — older pricing or account systems, plus spreadsheets and Airtable filling gaps. Common during an ERP migration; a frequent source of duplicate, drifting data.

Where the data lives — and where it breaks

The recurring pattern: each system holds one piece of the truth, and the value is in connecting them — but the connections are where things break.

A concrete failure mode: a tank monitor knows a customer is about to run dry, but that signal doesn't reach dispatch automatically, so the first anyone hears of it is an emergency phone call — converting a profitable planned full drop into a money-losing emergency run (Module 3). The data existed; the integration didn't.

Migration reality

Distributors often run mid-migration — moving from legacy systems and spreadsheets onto a unified ERP. During that window, data lives in two places, reconciliation is manual, and trust in any single number is low. This is messy but also the highest-leverage moment for getting the data architecture right, because habits and integrations formed now persist for years.

Where automation creates the most leverage

Map each lever to the integration that powers it:

  • Sourcing lever → pull OPIS/rack feeds into pricing automatically, so buyers always see the cheapest valid rack and quotes reflect real-time cost.
  • Routing lever → connect tank monitoring → dispatch, so deliveries are triggered by predicted run-dry dates and routed for density — replacing reactive emergency runs with planned full drops.
  • Pricing lever → integrate cost basis + delivery-cost model + tax tables into quoting, so every bid is built from real components (Module 5) instead of gut feel.
  • Cash lever → connect billing → accounting → A/R tracking, so invoices go out fast and slow-payers (Module 9) surface automatically.
  • Compliance layer → automate the tracking loops from Module 7 (tax tables, certificates, IFTA reconciliation, credential/inspection dates).

The unifying insight: in a thin-margin, logistics-heavy business, the profit isn't hiding in a better gallon — it's hiding in the gaps between systems. Closing those gaps is where technology turns the four levers into money. This is the bridge to Module 10's view on where AI specifically creates durable advantage.

Key terms
ERP
Enterprise Resource Planning — the system of record for accounts, pricing, contracts, inventory, and billing (e.g. iRely i21).
System of record
The authoritative source for a given piece of data; conflicts arise when multiple systems each claim to be it.
Dispatch / routing system
Software that turns orders into scheduled truck routes; where route density is operationally executed.
Tank monitoring (telemetry)
Sensors reporting customer tank fuel levels in near real time — the signal for predictive delivery.
Telematics
Truck-mounted GPS and sensor data: location, delivery confirmation, mileage (also feeds IFTA).
Data silo
A system whose data isn't connected to others, forcing manual re-keying and creating drift and delay.
Integration
Connecting systems so data flows automatically — the place most distributor value (and risk) actually sits.
